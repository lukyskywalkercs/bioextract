## FASE 1: PREPARACIÓN ANTES DE PUBLICAR

### 1.1. Verificación de Hechos
```
ANTES DE PUBLICAR, VERIFICAR:

✅ ¿Los datos del análisis son correctos?
✅ ¿Las referencias están citadas correctamente?
✅ ¿El tono es profesional y respetuoso?
✅ ¿Las críticas están justificadas técnicamente?
✅ ¿Se ofrecen soluciones constructivas?
```

### 1.2. Preparación de Evidencia
```
TENER LISTO:

1. Referencias técnicas:
   - TRIPOD checklist
   - Papers sobre validación en IA médica
   - Estudios comparativos con ViT

2. Ejemplos de buenas prácticas:
   - Papers que sí cumplen estándares
   - Ejemplos de validación externa

3. Código de ejemplo (si aplica):
   - Cómo implementar validación correcta
   - Cómo calcular métricas completas
```

---

## FASE 2: ESTRATEGIA DE PUBLICACIÓN

### 2.1. Orden de Publicación Recomendado

**Opción A: Comentario Directo (Más Diplomático)**
```
1. Comentar directamente en el post original
2. Tono muy constructivo y respetuoso
3. Ofrecer ayuda y colaboración
4. Esperar respuesta antes de publicar análisis completo
```

**Opción B: Post Independiente (Más Visible)**
```
1. Publicar análisis completo como post propio
2. Mencionar al autor original de forma respetuosa
3. Etiquetar si es apropiado
4. Compartir en grupos relevantes
```

**Opción C: Híbrido (Recomendado)**
```
1. Comentar brevemente en el post original
2. Publicar análisis completo como post propio
3. Enlazar ambos para contexto completo
```

### 2.2. Timing
```
MEJOR MOMENTO PARA PUBLICAR:

✅ Durante horas laborales (mayor visibilidad)
✅ Martes-Miércoles (mejor engagement)
✅ Evitar fines de semana
✅ Esperar 1-2 días después del post original (no inmediato)
```

## FASE 3: MANEJO DE RESPUESTAS

### 3.1. Respuesta Positiva del Autor

**Escenario:** El autor acepta las críticas y quiere mejorar
```
TU RESPUESTA:

"Excelente actitud científica. Estaría encantado de ayudar 
con las mejoras metodológicas. Podemos trabajar juntos en:

1. Implementar validación externa
2. Reportar métricas completas
3. Corregir problemas técnicos identificados

¿Tienen acceso a datasets externos para validación? 
Puedo ayudar con la implementación técnica."

OBJETIVO: Colaboración constructiva
```

### 3.2. Respuesta Defensiva del Autor

**Escenario:** El autor se defiende sin aceptar críticas
```
TU RESPUESTA (MANTENER PROFESIONALISMO):

"Entiendo tu perspectiva. Mi análisis se basa en estándares 
establecidos en IA médica (TRIPOD, validación externa, etc.).

Los puntos que menciono son:
- Recall perfecto es estadísticamente inusual
- Validación externa es estándar de oro
- Métricas completas son requeridas por journals

¿Podrías compartir más detalles sobre el dataset y 
metodología? Eso ayudaría a entender mejor los resultados."

OBJETIVO: Mantener diálogo técnico, no personal
```

### 3.3. Respuesta Agresiva

**Escenario:** El autor ataca personalmente
```
TU RESPUESTA (NO ENTRAR EN CONFLICTO):

"Mi análisis es puramente técnico y basado en estándares 
científicos establecidos. No es un ataque personal.

Si hay errores en mi análisis, estaré encantado de 
corregirlos con evidencia técnica.

El objetivo es mejorar la ciencia, no criticar personas."

OBJETIVO: Desescalar, mantener profesionalismo
```

### 3.4. Respuesta de la Comunidad

**Escenario:** Otros comentan apoyando o criticando
```
SI APOYAN TU ANÁLISIS:

"Gracias por el apoyo. El objetivo es mejorar los 
estándares en IA médica. La crítica constructiva es 
esencial para el avance científico."

SI CRITICAN TU ANÁLISIS:

"Gracias por el feedback. Si hay errores en mi análisis, 
por favor compártelos con evidencia técnica. Estoy 
abierto a aprender y corregir."

OBJETIVO: Mantener diálogo constructivo
```

## FASE 4: SEGUIMIENTO

### 4.1. Monitoreo de Respuestas
```
DURANTE PRIMERAS 24-48 HORAS:

- Responder a comentarios técnicos relevantes
- Aclarar dudas sobre el análisis
- Ofrecer ayuda si se solicita
- Mantener tono profesional
```

### 4.2. Actualización del Análisis
```
SI EL AUTOR COMPARTE MÁS INFORMACIÓN:

- Actualizar análisis con nueva información
- Reconocer información adicional
- Ajustar críticas si es necesario
- Mantener transparencia
```

- [Referencia técnica]
